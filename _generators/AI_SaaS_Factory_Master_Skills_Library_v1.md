## Standard Template for Every SKILLS.md

Each skill should contain:

- Mission
- Responsibilities
- Scope
- Industry standards
- Best practices
- Security considerations
- Quality checklist
- Common anti-patterns
- Acceptance criteria
- Collaboration rules
- Expected outputs
- Validation checklist
- Continuous improvement guidance

## Engineering Principles

Every specialist should follow:

- Security by Design
- Privacy by Design
- Accessibility by Default
- API First
- Cloud Native
- Observability First
- Test Automation
- Zero Trust
- Principle of Least Privilege
- Secure Defaults
- Semantic Versioning
- Documentation as Code
- Performance Budgets
- Backward Compatibility
- Idempotent Operations

---

## Long-term Vision

Build an AI Engineering Organisation rather than a single AI assistant.

Each specialist owns one domain and collaborates with other specialists to produce investor-grade SaaS products that are secure, scalable, maintainable, and commercially successful.

Future versions of this library should expand each specialist into its own dedicated SKILLS.md document with detailed responsibilities, checklists, standards, prompts, and validation rules.
